# 저장된 리포트 목록

`reports/` 디렉토리의 JSON 파일들을 읽어서 저장된 리포트 목록을 표시하세요.

## 조회 방법

1. `reports/` 디렉토리에서 모든 `.json` 파일을 찾으세요
2. 각 파일의 내용을 읽어서 파싱하세요
3. 아래 형식으로 보기 좋게 정리하세요

## 출력 형식

### 리포트가 있는 경우

```
## 저장된 리포트 ({N}개)

| # | 이름 | 설명 | 생성일 | 스케줄 |
|---|------|------|--------|--------|
| 1 | {name} | {description} | {created_at} | {schedule 요약 또는 "-"} |
| 2 | ... | ... | ... | ... |

### 상세 정보

**1. {리포트 이름}**
- 파일: `reports/{filename}.json`
- 조회 조건: {dimensions} × {metrics}
- 기간: {date_range}
- 스케줄: {frequency} {time} {day_of_week}

실행하려면: "'{리포트 이름}' 리포트 실행해줘"
스케줄 변경: "/smart-briefing:schedule {리포트 이름}"
```

### 리포트가 없는 경우

```
저장된 리포트가 없습니다.

GA 데이터에 대해 질문하면 분석 결과를 리포트로 저장할 수 있습니다.
예시:
- "이번 주 트래픽 소스별 성과 보여줘"
- "모바일 이탈률 분석해줘"
- "상위 페이지 성과 보여줘"

분석 후 "리포트로 저장해줘"라고 말씀하시면 됩니다.
```
